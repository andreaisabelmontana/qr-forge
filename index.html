QR Forge

A QR code looks trivial until you build the service behind one: encode arbitrary data, render it cleanly, customise it, and hand it back over a web API that thousands of clients can hit. QR Forge is that service, built on FastAPI. I rebuilt it from scratch because it's a perfect small backend — one clear job, done properly, from request to response.

PythonFastAPIREST PydanticOpenAPIDocker

What it is

A focused backend service, done right

QR Forge takes a payload — a URL, some text, contact details — and returns a QR code image, with options for size, error-correction level and styling. Simple on the surface, but the build is really about everything that makes a good API: typed request and response models, validation that rejects bad input before it causes trouble, sensible errors, and documentation that generates itself.

FastAPI is the natural home for it. You declare your data shapes once, and you get validation, serialisation and an interactive OpenAPI spec for free. The lesson is that a clean, well-typed contract is what turns a script into a service.

Architecture

How a request is served

Every call walks the same clean request-to-response path:

  1. Receive

    An HTTP request arrives with the payload and options.

  2. Validate

    Pydantic checks the input against the typed schema, rejecting bad data.

  3. Generate

    Encode the payload into a QR code at the requested settings.

  4. Respond

    Return the image (or a typed JSON payload) with the right content type.

  5. Document

    The same models keep the OpenAPI spec accurate automatically.

Reflection

What rebuilding it taught me

  • The contract is the product. Typed request and response models are what separate a robust service from a fragile script.
  • Validation belongs at the edge. Rejecting bad input the moment it arrives keeps the core logic clean and safe.
  • Docs should be free. Generating OpenAPI from the code means the documentation can never drift out of date.
  • Small scope, high polish. A service that does one thing flawlessly teaches more than a sprawling one that half-works.
  • Containerise early. "Works on my machine" stops being a problem the moment the service ships in a container.
